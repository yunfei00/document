# Markdown 编写帮助说明（速查 + 实战）

> 适用于：
> - GitHub / GitLab
> - 项目 README
> - 技术设计文档
> - 运维部署文档
> - 学习笔记 / 知识归档

---

## 1. Markdown 是什么？

Markdown（MD）是一种轻量级标记语言，特点是：

- 专注内容而不是排版
- 纯文本，适合版本控制
- 跨平台、通用性强
- 非常适合技术文档与交付文档

---

## 2. 标题（Heading）

```md
# 一级标题（文档标题）
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

规范建议：
- 一个文档只使用一个 `#`
- 正文从 `##` 开始
- 不要跳级

---

## 3. 段落与换行

```md
这是第一段内容。

这是第二段内容（中间空一行）。
```

强制换行（行尾两个空格）：

```md
这一行  
下一行
```

## 4. 字体样式

```md

## 8. 代码块（重点）

行内代码：
```md

```

```md
```bash

```
```

支持语言高亮：
`bash / python / json / yaml / sql / nginx / dockerfile`

## 16. 最佳实践

建议：
- 文件名使用英文 + 下划线
- 标题层级清晰
- 多用代码块，少截图
- 文档随版本维护

避免：
- 超长段落
- 标题层级混乱
- 图片无说明
