## ✅ 使用规则（强制执行 5 条）

1. **所有新内容先写到 Inbox 或直接写到对应目录**
2. **遇到问题必须产出一篇“解决方案文档”**（用模板）
3. **项目目录必须有 README 做导航**
4. **关键取舍写 ADR**（为什么这么设计）
5. **每周清理 Inbox**（把最重要的 3 条归档出去）

---

## 📌 推荐你的“工程资产优先级”（越上面越值钱）

1. 解决方案（可复制，立刻省时间）
2. 模板/清单（重复劳动归零）
3. 项目导航与交付文档（对外可交付）
4. ADR（多年后最值钱）
5. 技术结论（少而精）
