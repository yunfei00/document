# 1Panel 介绍与使用说明（通用实践版）

> 适用对象：开发 / 运维 / 测试 / 小团队管理员  
> 目标：快速了解 1Panel 能做什么，并能独立完成安装、基础配置、日常运维与常见排错。

---

## 0. 文档定位

### 本文档解决什么问题

- 1Panel 是什么、适合什么场景
- 如何在 Linux 服务器上快速安装
- 如何完成基础安全配置（账号、端口、防火墙、HTTPS）
- 如何做常见运维（站点、容器、数据库、备份）
- 出现常见错误时如何快速定位

### 本文档不解决什么

- 不覆盖所有高级特性（如复杂集群编排）

- 不替代生产变更流程与安全合规要求

## 1. 1Panel 是什么

1Panel 是一个面向 Linux 服务器的现代化运维管理面板，常用于：

- 网站与反向代理管理
- 容器与镜像管理（Docker）
- 数据库与中间件管理
- 文件、计划任务、日志、备份等运维功能

适合以下场景：

- 中小团队快速搭建并维护业务环境
- 个人或团队服务器统一管理入口
- 需要“可视化 + 命令行”协同运维的场景

---

## 2. 适用环境与前置条件

- 操作系统（建议）：Ubuntu 20.04+ / Debian 11+ / CentOS 7.9+ / Rocky Linux
- 服务器资源（建议起步）：2C4G（按业务量可上调）
- 网络要求：
  - 可访问面板管理端口
  - 如需在线安装应用，需可访问镜像源
- 权限要求：root 或具备 sudo 权限

安装前建议先做：

1. 更新系统补丁
2. 设置主机名与时区
3. 配置防火墙策略（仅开放必要端口）
4. 准备域名（若后续要启用 HTTPS）

## 3. 安装与首次登录（通用流程）

> 以下为通用安装流程。请始终以 1Panel 官方文档的最新命令与版本要求为准。

### 3.1 安装步骤（示意）

1. 登录服务器
2. 按官方文档执行安装脚本
3. 记录安装输出中的访问地址、初始账号信息或初始化提示

### 3.2 首次登录后必须做的事

1. 立即修改管理员密码（高强度）
2. 新建普通运维账号，避免长期使用超级管理员
3. 修改默认管理端口（非必要不暴露默认端口）
4. 配置访问白名单（若有固定办公出口 IP）
5. 开启 HTTPS（管理入口务必加密）

## 4. 基础安全加固清单（建议逐条完成）

- [ ] 管理密码长度 ≥ 12，包含大小写、数字、符号
- [ ] 禁止弱口令与共享账号
- [ ] 管理端口不对公网全开放（优先内网/VPN）
- [ ] 启用防火墙，仅开放业务端口（80/443/必要业务端口）
- [ ] 配置自动备份，并定期演练恢复
- [ ] 服务器开启时间同步（NTP）
- [ ] 宿主机 SSH 关闭密码登录，改用密钥（推荐）
- [ ] 定期升级系统与 1Panel（先测试后生产）

## 5. 常用功能上手

### 5.1 网站与反向代理

典型步骤：

1. 新建站点（绑定域名）
2. 配置反向代理到后端服务（如 `127.0.0.1:8000`）
3. 申请/导入 SSL 证书
4. 开启 HTTPS 强制跳转
5. 配置访问日志与错误日志路径

建议：

- 生产环境统一走 443
- 对后台路径增加访问控制
- 大文件上传场景要同步调高 Nginx 上传限制

### 5.2 容器管理（Docker）

常见操作：

- 拉取镜像并创建容器
- 绑定端口、挂载目录、设置环境变量
- 查看实时日志与资源占用
- 重启、滚动更新、回滚到稳定镜像

建议：

- 重要服务使用固定版本标签，不直接用 `latest`
- 数据目录统一挂载到宿主机并纳入备份
- 先在测试环境验证再升级生产

### 5.3 数据库管理

常见操作：

- 创建数据库与账号
- 按最小权限原则授权
- 开启定期备份与保留策略
- 慢查询排查与索引优化（结合业务）

建议：

- 数据库不直接暴露公网
- 高风险操作（删库、结构变更）走变更审批

### 5.4 计划任务与备份

建议至少配置三类任务：

1. 数据库备份（每天）
2. 应用数据目录备份（每天/每周）
3. 系统日志归档与清理（按容量策略）

备份策略建议：

- 本地保留 + 异地/对象存储副本
- 设定保留周期（如 7/30/90 天分层）
- 每月至少一次恢复演练

## 6. 运维排错速查

### 6.1 面板无法访问

排查顺序：

1. 服务是否运行（面板服务状态）
2. 管理端口是否监听
3. 防火墙/安全组是否放行
4. 服务器是否有网络连通性问题
5. 是否被反向代理或 WAF 拦截

### 6.2 网站 502 / 504

排查顺序：

1. 后端服务是否存活
2. 反向代理 upstream 地址与端口是否正确
3. 后端日志是否有报错
4. 超时参数是否过小
5. 服务器资源是否耗尽（CPU/MEM/磁盘）

### 6.3 证书申请失败

常见原因：

- 域名解析未生效
- 80/443 被占用或未放行
- DNS 配置不一致
- 频繁申请触发限流

### 6.4 容器频繁重启

常见原因：

- 环境变量缺失
- 挂载目录权限错误
- 启动命令或健康检查失败
- 依赖服务未就绪

## 7. 升级与回滚建议

升级前：

1. 先备份（配置、数据、镜像）
2. 在测试环境验证版本兼容性
3. 明确回滚点（镜像 tag / 配置快照 / 数据备份）

升级中：

- 记录操作步骤与时间点
- 观察关键指标（可用性、延迟、错误率）

升级后：

- 验证核心业务链路
- 保留旧版本可快速回退
- 形成升级记录文档
